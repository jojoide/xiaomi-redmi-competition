📊 About This Dashboard

This is a competitive price intelligence tool for Xiaomi & Redmi Mexico, tracking the weekly retail prices of 48 SKUs across five brands — Xiaomi, Samsung, Honor, Motorola, and OPPO — in the Mexican market. Data is collected twice a week (Monday and Thursday) from W23 onward. All data is embedded directly in the HTML file; no internet connection or server is needed to view it.

📄 Tab Guide

Recent

A brand × price-tier matrix showing the current week's prices for every tracked SKU. Rows are anchored to Xiaomi models sorted by price within each segment. Red highlight = a competitor is cheaper than Xiaomi in that row. Badges show week-over-week change and MSRP delta. Use the Segment chips to filter by Entry / Note / High.

Historical Data

A swimlane trend chart showing price movement over time. Filter by segment, brand, and week range using the controls above the chart. Dashed lines indicate weeks where no price was recorded for that SKU.

⚙️ First-Time Setup

Do this once before running the update script for the first time.

1
Install Python 3.9 or later

Download from python.org. During installation, check "Add Python to PATH". Verify by running python --version in Command Prompt.

2
Install Python dependencies

Open Command Prompt and run:
pip install requests beautifulsoup4 openpyxl pandas playwright

3
Install the Chromium browser for Playwright

Still in Command Prompt, run:
python -m playwright install chromium
This downloads a headless browser (~130 MB) used to fetch prices from telcel.com, which blocks regular HTTP requests.

4
Verify the setup with a dry run

Navigate to the project folder in Command Prompt and run:
cd "C:\Users\...\Xiaomi & Redmi Competition"
python update_prices.py --dry-run
You should see prices auto-fetched for most SKUs. Nothing gets saved in dry-run mode.

⚠ Always use Command Prompt (cmd.exe), not PowerShell. PowerShell has encoding issues with the Chinese characters in the script.

🔄 Weekly Update — Monday & Thursday

1
Open Command Prompt and navigate to the folder

cd "C:\Users\...\Xiaomi & Redmi Competition"

2
Run the update script

On Monday: python update_prices.py --monday
On Thursday: python update_prices.py --thursday
The script labels the week automatically (e.g. W23-M or W23-T).

3
Wait for auto-fetch to complete

The script launches a headless browser and fetches prices from telcel.com for all ~43 Telcel SKUs, plus mi.com and coppel.com for the remaining 2. This takes about 1–2 minutes. You'll see [OK] for each successful fetch and [--] for failures.

4
Enter prices for any failed fetches

After auto-fetch, the script prompts you only for SKUs it couldn't fetch automatically. Open the product URL shown, check the price, and type it in. Enter s to skip a SKU if it's out of stock or delisted.

5
Check the dashboard

Refresh index.html in your browser. The Recent tab should now show the new week label and updated prices. The Historical chart will also include the new data point.

6
If something looks wrong — revert

To erase a week's data and roll back the dashboard:
python update_prices.py --revert W23-M
Replace W23-M with the label you want to remove. This deletes those rows from historical_data.xlsx and regenerates the dashboard.

+ Adding or Removing SKUs

To add a new model

Open SKUS.xlsx and add a new row with these columns:

  • 档位 — segment: 入门 (Entry), Note, or 高端 (High)
  • 品牌 — brand in Chinese: 小米, 三星, 荣耀, 摩托罗拉, or OPPO
  • 机型 — model name (e.g. Galaxy A37 5G)
  • 配置 — memory config (e.g. 8+256)
  • 链接 — product URL (telcel.com, mi.com, or coppel.com)

Also add an MSRP row in the MSRP sheet of historical_data.xlsx with the same 档位 / 品牌 / 机型 / 配置 columns plus the MSRP price.

To remove a model

Delete its row from SKUS.xlsx. Its historical data remains in historical_data.xlsx and will still appear in the Historical chart, but it won't be collected in future updates.

🔧 Troubleshooting

Script exits silently after "loading constants..."

The main() function wasn't called — this was a known bug that has been fixed. Make sure you're running the latest version of update_prices.py.

Most fetches fail / show [--]

Playwright may not be installed. Run python -m playwright install chromium. If already installed, telcel.com may be blocking the headless browser — try re-running; it usually succeeds on retry.

Script crashes with encoding error

You're running in PowerShell. Switch to Command Prompt (cmd.exe) instead.

Dashboard shows old data after update

Hard-refresh the browser: Ctrl + Shift + R (Windows) or Cmd + Shift + R (Mac). The HTML file is locally cached by some browsers.

historical_data.xlsx won't open / is corrupt

The file may not have fully synced from iCloud. Wait a few seconds and try again. If the issue persists on the update script side, the script has a built-in EOCD repair that handles this automatically.

Week label conflict ("week already in dataset")

You're running the script for a week that already has data. The script will ask if you want to overwrite — type y to replace, or n to abort.

⚙️ 首次配置

首次运行更新脚本前，请完成以下步骤。

1
安装 Python 3.9 或更新版本

从 python.org 下载安装。安装时勾选 "Add Python to PATH"。通过运行 python --version 验证。

2
安装 Python 依赖包

打开命令提示符，运行：
pip install requests beautifulsoup4 openpyxl pandas playwright

3
安装 Playwright Chromium 浏览器

在命令提示符中运行：
python -m playwright install chromium
这将下载一个无头浏览器（约 130 MB），用于从 telcel.com 获取价格（该网站会屏蔽普通 HTTP 请求）。

4
通过演习运行验证配置

在命令提示符中进入项目文件夹，运行：
cd "C:\Users\...\Xiaomi & Redmi Competition"
python update_prices.py --dry-run
大多数 SKU 应能自动获取价格。演习模式不保存任何数据。

⚠ 请始终使用命令提示符（cmd.exe），而非 PowerShell。PowerShell 对脚本中的中文字符存在编码问题。

🔄 每周更新 — 周一 & 周四

1
打开命令提示符，进入项目文件夹

cd "C:\Users\...\Xiaomi & Redmi Competition"

2
运行更新脚本

周一运行：python update_prices.py --monday
周四运行：python update_prices.py --thursday
脚本会自动生成周标签（如 W23-M 或 W23-T）。

3
等待自动采集完成

脚本启动无头浏览器，依次从 telcel.com（约 43 个 SKU）、mi.com 和 coppel.com 获取价格。耗时约 1—2 分钟。采集成功显示 [OK]，失败显示 [--]。

4
手动输入采集失败的价格

自动采集完成后，脚本仅提示无法自动获取的 SKU。打开显示的商品链接，查询价格并输入。输入 s 可跳过已下架或缺货的机型。

5
检查仪表盘

在浏览器中刷新 index.html，最新价格标签页应显示新的周标签和更新后的价格，历史数据图表也会新增数据点。

6
如有问题——回退

如需删除某周数据并回滚仪表盘：
python update_prices.py --revert W23-M
将 W23-M 替换为需要删除的标签。该命令会从 historical_data.xlsx 删除相关行并重新生成仪表盘。

+ 新增或删除追踪机型

新增机型

打开 SKUS.xlsx，添加包含以下列的新行：

  • 档位 — 入门、Note 或 高端
  • 品牌 — 品牌中文名：小米、三星、荣耀、摩托罗拉 或 OPPO
  • 机型 — 型号名称（如 Galaxy A37 5G）
  • 配置 — 内存配置（如 8+256）
  • 链接 — 商品链接（telcel.com、mi.com 或 coppel.com）

同时在 historical_data.xlsx 的 MSRP 工作表中添加对应行（档位 / 品牌 / 机型 / 配置 + MSRP 价格）。

删除机型

从 SKUS.xlsx 删除对应行。历史数据仍保留在 historical_data.xlsx 中并显示于历史图表，但不再参与后续更新采集。

🔧 常见问题排查

脚本在"loading constants..."后静默退出

main() 函数未被调用——这是已修复的已知 bug。请确保使用最新版 update_prices.py。

大多数价格采集失败 / 显示 [--]

可能是 Playwright 未安装。运行 python -m playwright install chromium。若已安装，telcel.com 可能暂时屏蔽了无头浏览器——重新运行通常可解决。

脚本崩溃并报编码错误

你在使用 PowerShell。请切换至命令提示符（cmd.exe）。

更新后仪表盘仍显示旧数据

强制刷新浏览器：Windows 按 Ctrl + Shift + R，Mac 按 Cmd + Shift + R。

historical_data.xlsx 无法打开 / 文件损坏

文件可能未从 iCloud 完全同步。稍等片刻后重试。脚本内置了 EOCD 修复机制，可自动处理大多数同步问题。

周标签冲突（"week already in dataset"）

该周数据已存在。脚本会询问是否覆盖——输入 y 覆盖，输入 n 中止。
